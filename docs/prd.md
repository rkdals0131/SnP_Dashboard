# PRD — S\&P 리스크·신호 집계판 + 확률 콘 대시보드 v0.4.1

문서 소유자: 사용자(강민 김)
연관 문서: dev-architecture.md — 모듈화·검증 통합 v0.4.1

## v0.4.1 Lean 조정(요약)

* 데이터 빈티지: 초기 버전은 ALFRED 연동을 보류하고, 지표별 **발표 지연 규칙 기반 시뮬레이션**으로 선견편향을 억제합니다(예: 월간 지표는 다음 달 초부터 사용 가능).
* 실행 진입점: `scripts/` 제거, `src/cli.py`에 Typer 기반 **단일 CLI**를 두고 `Makefile`에서 이를 호출합니다.
* 테스트 전략: **E2E(UI) 테스트 제외**, 유닛·통합 테스트에 집중합니다.
* 기여도: M1에서는 단순화(계수×Δ특징량, 상위 변화량 Top-N)로 시작하고, SHAP 등은 확장 노트로 이동.

편집 원칙: 투자조언 아님, 연구·관측용. 굵은 글씨 지양, 한국어 표기 일관.

# 1. 상황·배경·맥락

* 질문의 핵심: 미장 S\&P 500 지수가 오를까, 내릴까. 단정이 아닌 확률적 범위로 답한다.
* 환경 제약: 블룸버그 단말기 없음 → 공공 API 기반(무료 위주), 로컬 구동 전제.
* 커뮤니케이션 철학: 현재 상태가 향후 분포의 중심·폭·꼬리에 주는 압력의 부호와 크기를 투명하게 시각화.

# 2. 문제 정의

* 알고 싶은 것

  * A 탭: 거시·금융 신호를 레벨·모멘텀·가속도로 표준화하여 지금이 좋다/나쁘다, 상승/하락 중인지 요약
  * B 탭: 같은 상태에서 1·3·6·12개월 S\&P 수익률의 조건부분포를 팬 차트로 제시
* 난점: 발표 지연, 빈도 혼합(일·주·월·분기), 데이터 정정, 레짐 변화, 낮은 신호대잡음비
* 원칙: 방향 단정 금지, 분포·범위 중심의 확률적 커뮤니케이션

# 3. 목표와 비전

* 한 화면에서 신호 집계판과 확률 콘을 제공하여, 현재 상태와 전망을 빠르게 연결
* 정량 목표

  * 일간 지표 반영 지연 1영업일 이내(공공 API 기준)
  * 시나리오 조정 시 재계산 2초 이내
  * 예측구간 적중률의 캘리브레이션 유지(예: 80% 구간 75\~85%)

# 4. 탭 구조

## 탭 A. 매크로 시황 요약(신호 집계판)

목표

* 지표를 표준화하고 합성 점수와 브레드스를 산출, 사용자가 30\~60초 내 시황을 구술 가능

구성 요소

* 합성 점수 타일: MacroScore(−1\~+1), Momentum, Acceleration
* 브레드스 바: 긍정 신호 비중, 상·하위 기여 지표 3종
* 레짐 배지: 금융여건(완화/중립/긴축), 경기모멘텀(확장/둔화), 물가모멘텀(상승/하락)
* 지표 카드: 물가, 성장, 고용, 금리·수익률곡선, 금융여건, 크레딧, 변동성, 원자재
* 시황 자동요약(변화 감지 중심): 가장 큰 변화 Top-N 사실을 우선 표기(예: "NFCI가 10년 분위 기준 +15%p로 긴축 구간 진입"). 결론 단정 대신 정황·가능성 표현 사용.

스코어 산식(초안)

* 레벨 스코어: 10년 분위 백분위로 \[−1,+1] 선형 변환
* 모멘텀: 1차차분의 z-score, 가속도: 2차차분의 z-score
* 카테고리 합성: wL·레벨 + wM·모멘텀 + wA·가속도 (기본 0.5/0.35/0.15)
* 매크로 합성: 카테고리 가중 평균(기본 동일가중)

판정 로직(0.4.1)

* 기본 표기: 브레드스 중심. 상위 분위 구간(예: 상위 50%·상위 20%)에 위치한 지표 비중과 그 변화(ΔMacroBreadth)를 핵심 노출.
* 상승/하락 표기: Momentum 임계치 대신 ΔMacroBreadth 절대 변화(예: ±10%p) 기준.
* MacroScore 임계치(±0.5)는 보조지표로 툴팁 제공(기본 UI 비노출). 경고 배지(수익률곡선 역전 심화, 콘 폭 급확대 등)는 유지.

## 탭 B. S\&P 500 팬 차트

목표

* 조건부분포 $P(r_{t\to t+h} \mid \text{상태변수}_t)$를 분위수로 제시, 불확실성과 중심 이동을 추적

구성 요소

* 가격 차트 + 팬 차트 오버레이(5·10·25·50·75·90·95%)
* 불확실성 게이지: 90%−10% 분위 폭
* 비교 토글: 과거 시점 예측분포 vs 실현 경로
* 시나리오 슬라이더: 10Y, 3M, VIX, WTI, NFCI, CPI 서프라이즈
* 기여도 카드(0.4.1): M1은 단순화(선형 계수×Δ특징량, 또는 변화량 Top-3/Bottom-3). 정교한 설명(예: SHAP)은 후속 버전으로 이월.

추정 원리(요약)

* 단기(1M)는 시장축 가중(VIX·SKEW 등), 중기(3\~12M)는 거시축 가중(NFCI·스프레드 등)
* 모델은 분위회귀를 기본으로 하되 단기 폭은 VIX로 앵커, 중기는 거시 신호로 형상 결정

# 5. 검증·백테스트 정성 가이드

목적

* 사용자에게 모델의 신뢰도와 한계를 명확히 전달

원칙

* 과신 금지: 가능성·정황·시사 표현 사용
* 투명성: 예측 시점의 데이터 버전과 가용 범위를 표시
* 시각 검증: 과거 t0 시점의 팬 차트와 그 이후 실현 경로 오버레이 제공
* 정량 검증 결과는 dev-architecture.md의 규격에 따르되, 본 탭에는 핵심 요약만 표시

지표 예시

* 예측구간 적중률 캘린더(50%, 80% 구간)
* 기준선 대비 개선 배지(평균수익률·랜덤워크·옵션단독)

문구 가이드

* 예시: 현재 신호는 둔화 가능성을 시사합니다. 하방 꼬리의 확률 질량이 소폭 확대되었습니다.

# 6. 데이터 소스 및 신호 목록(시즌 1)

* BLS: UNRATE 등 노동시장 핵심 시계열
* BEA: PCE, GDP 관련
* 연준/연구기관: 시카고 연은 NFCI, 필라델피아 연은 ADS, 애틀랜타 연은 GDPNow
* 옵션 데이터: MVP는 VIX·SKEW, 심화는 옵션체인

# 7. 시스템 개요(요약)

* 로컬 우선. 수집층 → 정합·피처층 → 모델층 → 시각화층. 빈티지 관리 가능 시 ALFRED 사용
* 세부 아키텍처와 코드 규약은 dev-architecture.md에 따름

# 8. 기능 요구사항

* 탭 A: 합성 점수, 브레드스, 레짐 배지, 카드 8종, 자동요약
* 탭 B: 1·3·6·12개월 팬 차트, 게이지, 비교 토글, 시나리오, 기여도 카드
* 검증 요약: 적중률·캘리브레이션 요약, 기준선 대비 배지

# 9. 비기능 요구사항

* 성능: 시나리오 재계산 2초 이내, 초기 로드 3초 이내
* 신뢰성: 수집 재시도 3회, 캐시 TTL 5분
* 재현성: 모델 버전 고정, 스냅샷 데이터 보관
* 보안·라이선스: 공공 데이터 ToS 준수, 출처 표기

# 10. KPI·수용 기준

* 카드 10개 이상 레벨·Δ·Δ²·역사 분위 표시
* 브레드스 표기: 상위 50% 구간 지표 비중과 전월 대비 변화(±10%p 이상) 탐지·표시
* 1·3·6·12M 팬 차트 전 구간 생성, 시나리오 3종 이상 즉시 반영
* 80% 구간 적중률 75~85%, 50% 구간 45~55%

# 11. 릴리즈 계획

* M1 PoC: FRED 중심 수집, 카드 6종, 1M 팬 차트, 시나리오 3종, 검증 요약 탭
* M2 확장: 3·6·12M 수평, 기여도 카드, 검증 히스토리
* M3 안정화: 빈티지 백테스트, 알림, 내보내기

# 12. 리스크와 대응

* 데이터 지연·중단: 캐시·대체 소스
* 레짐 변화: 롤링 학습·윈도 탐색
